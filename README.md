# Enterprise SIEM Lab using Splunk, Suricata & Syslog

## Overview

This project presents the design and implementation of a Security Information and Event Management (SIEM) solution for centralized log collection, threat detection, monitoring, and incident response.

The environment integrates **Suricata IDS**, **Syslog (rsyslog)**, and **Splunk Enterprise** to collect, forward, index, and analyze security events in real time. To improve availability and fault tolerance, the Splunk deployment was extended into an **Indexer Cluster** consisting of one Cluster Manager and three Peer Indexers.

This project was developed as my graduation project (PFE) in Cybersecurity.

## Attack Simulation

The SIEM solution was validated through a complete attack scenario performed from a Kali Linux machine.

### Reconnaissance

* Network discovery
* Port scanning using Nmap and Zenmap

### Initial Access

* SSH brute-force attack using Hydra

### Privilege Escalation

* Verification of sudo privileges

### Persistence

* Creation of a privileged user

### Covering Tracks

* Removal of authentication and system logs

---

## Detection

The SIEM successfully detected:

* Ping activity
* Port scanning
* SSH brute-force attacks
* User account creation
* Group creation
* Security events generated by Suricata
* System log events
* Suspicious network activity

Splunk generated alerts in real time and sent notifications through the Splunk Mobile application.

---

## Security Hardening

After the attack simulation, several defensive measures were implemented:

* Password strengthening
* Removal of unauthorized users
* Firewall IP blocking
* SSH hardening
* Disable root login
* Key-based SSH authentication
* SSH port modification
* Fail2Ban deployment
